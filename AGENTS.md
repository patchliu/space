# Repository Instructions

This repository stores lightweight Markdown resources.

When adding or editing content:

1. Keep the top level stable: `.github/`, `assets/`, `prompts/`, `skills/`, `knowledge/`.
2. Use short, lowercase, hyphenated filenames.
3. Keep documentation focused on repository structure.
4. Keep assets referenced by docs under `assets/`.
5. Avoid adding new top-level directories.
